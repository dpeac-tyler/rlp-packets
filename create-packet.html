Packets group a primary application type with one or more secondary application types. When a packet is created, the primary applicant completes the primary application, after which affiliated users are prompted to complete the associated secondary applications. Each secondary can be marked as Required, meaning it must be completed before the packet can be submitted or Optional, allowing it to be skipped. At least one required secondary must be defined before a packet can be saved.
